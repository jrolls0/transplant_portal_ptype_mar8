# Frontend Design

Implement the chosen visual direction with a strong point of view.

## Rules
- Commit to one direction instead of mixing styles.
- Use typography and spacing to create hierarchy.
- Prefer a few strong layout decisions over many decorative embellishments.
- Reuse existing components where possible.
- Create only the smallest number of new primitives needed.
- Use realistic copy and believable sample data.
- Make the UI responsive and coherent.
- Avoid default-looking Tailwind/shadcn compositions.
- After implementation, review what still looks generic and refine it.
