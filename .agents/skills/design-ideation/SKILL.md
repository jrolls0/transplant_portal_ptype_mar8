# Design Ideation

Your job is to do the design thinking before implementation.

## Process
1. Inspect the current UI, screenshots, nearby code, and product context.
2. Do not write code first.
3. Produce exactly 3 distinct visual directions.
4. For each direction, provide:
   - a short name
   - the visual philosophy
   - typography strategy
   - spacing / density approach
   - color strategy
   - component style
   - interaction / motion style
   - why it fits this product and user
   - implementation risk
5. Make the directions meaningfully different from each other.
6. Avoid generic AI-dashboard aesthetics unless the product truly calls for them.
7. Rank the 3 directions and recommend one.
